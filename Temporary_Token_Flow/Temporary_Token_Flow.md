# Temporary Token Flow — разбор нового функционала 
## Смысл фичи:
 Админ может создать временный токен для человека без регистрации. Юзер получает ссылку, переходит по ней — и попадает в платформу с ограниченным доступом на основе роли. Как гостевой доступ 

## Два метода
 
**GenerateToken** — создаёт токен  
**AuthorizeWithToken** — авторизует по токену

### Шаг 1 — GenerateToken
Админ передаёт:
1. данные временного пользователя (имя, фамилия, код страны, эмейл, телефон, пароль)
2. ID роли которую дать
3. на сколько минут действует доступ (в минутах)
 
Система генерирует случайную строку из 64 байт → переводит в base64 → сохраняет в базу вместе с метаданными в JSON. Возвращает токен-строку 

### Шаг 2 — AuthorizeWithToken
 
Человек переходит по ссылке с токеном. Система:
 
1. Ищет токен в базе — нет или уже использован возвращает 400
2. Десериализует metadata (достаёт данные пользователя из JSON)
3. Смотрит: есть ли уже такой юзер в базе по email?
 
**Если юзера нет (первый вход):**
1. создаёт пользователя через `CreateAsync`
2. запускает таймер — `ActivatedOn = now`
 
**Если юзер уже есть (повторный вход в рамках времени):**
1. проверяет не истёк ли токен (`ActivatedOn + ExpiryInMinutes > now`)
2. если истёк → `IsUsed = true` → 400 "Token is obsolete"
 
4. Помечает токен как использованный `IsUsed = true`
5. **Критически важно:** перечитывает юзера из базы заново — чтобы EF Core подтянул все связи (роль, клеймы и тд). Без этого был 500
6. Генерирует JWT + refresh токен → возвращает
 
## Что важно учесть при тестировании
 
1. Email должен быть реальным форматом — `test@test.com`, не просто `test` (попался несколько раз на это)
2. Телефон должен быть в формате `+998971112389` — есть валидатор
3. Пароль должен быть сложным — `Test123!` (заглавная + цифра + спецсимвол). А обычный `test` не пройдёт Identity валидацию 
4. Токен одноразовый — после первого использования `IsUsed = true`, второй раз 400
 
## Баг который мы нашли в процессе
 
Изначально после `CreateAsync` юзер сразу передавался в `CreateToken`. Но объект был "сырой" — EF Core не подгрузил связанные данные (роль, RoleControllerActions). При попытке собрать клеймы — 500
 
Роман пофиксил: после создания юзера делается отдельный `GetAsync` из базы. Теперь объект полный и клеймы строятся корректно
