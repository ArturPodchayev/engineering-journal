## над чем работал

**литкод**

неделя была выходная, но стрик не ломал — щёлкал задачи каждый день усердно, прокачивал алгоритмы и структуры данных репо `leet-dominator` пополнялся: структура одна папка на задачу, все варианты решений внутри

**github contribution graph**

обнаружил что 15+ файлов в день считались как 6 контрибуций — причина в том что использовал кнопку "add file" через web ui, github группирует такие действия и занижает count исправил паттерн: теперь один осмысленный коммит на одну задачу, всё через локальный git — graph сразу выровнялся

**прогон по инсайту**

прошёлся по всему что делал за последние недели — освежил в голове архитектуру, повторил механику миграций ef core, пробежался по rabbitmq/masstransit которые тимлид давал изучить заодно разобрал несколько мелких моментов в коде которые раньше принимал на веру

**настройка cursor**

настроил cursor под все три проекта — tonsense, insight и portfolio создал `.cursor/rules/general.mdc` и `.cursorignore` для каждого, подключил mcp серверы (notion, miro, figma, vercel), выставил нужные модели и включил privacy mode

исправил несоответствие в правилах — была указана версия next.js 14, по факту 16.2.4

## техническое решение недели

github считает web ui "add file" действия как меньше контрибуций чем они есть на самом деле локальные коммиты через git считаются честно — один коммит, одна единица в графе простая вещь но незаметно занижала всю статистику несколько недель

## что выучил
