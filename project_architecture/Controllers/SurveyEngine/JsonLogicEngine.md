## JsonLogicEngine

# JsonLogicEngine - мозг движка опросов, его главная задача - это решить куда идти после того как пользователь ответил на вопрос. Он читает правила из Survey JSON и вычисляет следующий шаг 
Он получает схему опроса, текущий шаг и переменные пользователя — и возвращает ID следующего шага. Сам не знает ничего про опрос, просто читает правила и вычисляет результат

## Интерфейс IJsonLogicEngine

Три метода — три задачи:

- `EvaluateAsync` — вычисляет результат правила. Берёт правило и данные пользователя, возвращает `true` или `false`
- `ValidateRule` — проверяет корректность синтаксиса правила, правильно ли оно написано
- `GetNextStepAsync` — главный метод. Принимает схему, текущий шаг и переменные, возвращает ID следующего шага

## ParsedRule — контейнер результата парсинга

Не место где парсим — а папка где храним результат:

```csharp
public class ParsedRule
{
    public JsonElement Rule { get; set; }    // само правило после парсинга
    public int Complexity { get; set; }      // сложность правила — нужна для выбора стратегии
    public DateTime ParsedAt { get; set; }   // когда было распарсено
}
```

## Выбор стратегии по сложности

Движок сам выбирает как вычислять правило в зависимости от его сложности — как выбор маршрута доставки:

```csharp
return parsed.Complexity switch
{
    < 3  => EvaluateSimple(parsed.Rule, data.RootElement),      // простое — быстрый прямой метод
    < 10 => EvaluateCompiled(parsed.Rule, data.RootElement),    // среднее — компиляция
    _    => EvaluateInterpreted(parsed.Rule, data.RootElement)  // сложное — интерпретация с кэшем
};
```

Зачем так? Чтобы не тратить ресурсы на простые правила — лёгкое вычисляется быстрым методом, сложное — мощным но медленным

## GetNextStepAsync — как определяется следующий шаг

Два случая в зависимости от типа поля `next` в схеме:

**Случай 1 — простой, next это строка:**
```json
"next": "feedback"
```
Следующий шаг фиксирован — просто возвращаем его

**Случай 2 — сложный, next это объект с правилами:**
```json
"next": {
  "if": [
    [{ "==": [{ "var": "score" }, "1"] }, "low_step"],
    [{ ">=": [{ "var": "score" }, "4"] }, "high_step"]
  ],
  "else": "feedback"
}
```
Перебираем правила сверху вниз — возвращаем первый шаг у которого условие `true`. Если ни одно не сработало — идём в `else`

Если шаг вообще не найден — возвращаем `"end"` как fallback
